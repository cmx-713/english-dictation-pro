# 数据未保存问题排查

## 🔍 步骤 1: 检查浏览器控制台

1. **打开网站**
2. **按 F12 打开开发者工具**
3. **切换到 Console 标签**
4. **完成一次练习**
5. **查看是否有错误信息**

### 常见错误信息：

#### 错误 1: 环境变量未配置
```
缺少 Supabase 环境变量，后台数据保存功能将不可用。
```
**解决：** 需要在 Netlify 配置环境变量

#### 错误 2: 插入失败
```
同步到云端失败: {...}
```
**解决：** 查看具体错误信息

#### 错误 3: 网络错误
```
Failed to fetch
```
**解决：** 检查网络连接

---

## 🔍 步骤 2: 检查 Netlify 环境变量

1. **登录 Netlify Dashboard**
2. **选择你的项目**
3. **点击 "Site settings"**
4. **点击左侧 "Environment variables"**
5. **检查是否有以下变量：**

```
VITE_SUPABASE_URL = https://xxx.supabase.co
VITE_SUPABASE_ANON_KEY = eyJhbGc...一长串
```

### 如何获取这两个值？

1. **打开 Supabase Dashboard**
2. **选择你的项目**
3. **点击左侧 "Project Settings" (齿轮图标)**
4. **点击 "API"**
5. **复制：**
   - Project URL → `VITE_SUPABASE_URL`
   - anon public key → `VITE_SUPABASE_ANON_KEY`

---

## 🔍 步骤 3: 检查本地测试

如果本地开发环境：

1. **创建 `.env.local` 文件**（在项目根目录）
2. **添加内容：**
```
VITE_SUPABASE_URL=你的项目URL
VITE_SUPABASE_ANON_KEY=你的匿名密钥
```
3. **重启开发服务器**
```bash
npm run dev
```

## ✅ 最可能的原因

### 1. 环境变量未配置（最常见）

**症状：** 控制台显示 "缺少 Supabase 环境变量"

**解决：**
- 在 Netlify 添加环境变量
- 重新部署

### 2. 密钥错误

**症状：** 控制台显示 "Invalid API key" 或 401 错误

**解决：**
- 检查 ANON_KEY 是否复制完整
- 确认是 anon public key，不是 service_role key

### 3. 表名或字段名不匹配

**症状：** 控制台显示 "column does not exist"

**解决：**
- 确认表名是 `practice_records`
- 确认所有字段都已创建

---

## 🚀 快速修复

如果是 Netlify 部署的应用：

1. **配置环境变量**
   - Netlify → Site settings → Environment variables
   - 添加两个变量
   - 点击 "Save"

2. **触发重新部署**
   - Netlify → Deploys
   - 点击 "Trigger deploy" → "Deploy site"

3. **等待部署完成**（2-3分钟）

4. **重新测试**
