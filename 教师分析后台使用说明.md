# 教师分析后台 - 使用说明

## ✨ 功能概览

全新的教师分析后台提供了强大的数据分析和可视化功能，帮助教师全面掌握学生的学习情况。

---

## 🚀 如何访问

### **方法 1: URL 参数**
在网址后添加 `?mode=teacher`

```
https://你的网站.netlify.app/?mode=teacher
```

### **方法 2: 代码中添加入口（可选）**
可以在 Header 组件中添加一个"教师后台"按钮

## 📊 功能详解

### 1️⃣ **总览标签页**

#### **统计卡片**
展示关键指标：
- 📌 **总学生数** - 所有注册学生，显示本周活跃人数
- 📌 **总练习次数** - 累计练习次数和人均练习
- 📌 **平均正确率** - 全体学生的平均表现
- 📌 **班级数量** - 有数据的班级总数

#### **数据图表**
- **📈 最近7天练习趋势**
  - 双线图：练习次数 + 平均正确率
  - 了解学习活跃度变化
  
- **🥧 难度等级分布**
  - 饼图显示各难度级别的练习分布
  - 了解学生选择的素材难度

---

### 2️⃣ **学生标签页**

#### **功能特点**
- ✅ 按班级筛选学生
- ✅ 学生排行榜（按平均正确率排序）
- ✅ 前三名特殊标识（金、银、铜）
- ✅ 点击展开查看详情

#### **学生卡片信息**
**基础信息：**
- 姓名和班级
- 练习次数
- 总单词数
- 完美句数
- 平均正确率

**展开详情：**
- 最高正确率
- 最低正确率
- 本周练习次数
- 最后练习时间

---

### 3️⃣ **班级标签页**

#### **班级卡片展示**
每个班级显示：
- 📚 班级名称
- 👥 学生人数
- 📊 班级平均正确率
- 📝 总练习次数
- 📖 总单词数
- 🎯 人均练习次数

#### **应用场景**
- 对比不同班级的学习情况
- 发现优秀班级和需要帮助的班级
- 班级间良性竞争

---

### 4️⃣ **趋势标签页**

#### **三大分析图表**

**📊 练习活跃度趋势**
- 柱状图：每日练习次数 + 活跃学生数
- 了解学生参与度

**📈 正确率趋势**
- 折线图：每日平均正确率变化
- 评估学习效果提升

**📉 难度级别分析**
- 柱状图：各难度的练习次数和正确率
- 评估难度设置是否合理

---

## 🎯 典型使用场景

### **场景 1: 每周教学总结**

```
1. 打开"总览"标签页
   → 查看本周总体数据
   
2. 切换到"趋势"标签页
   → 查看本周练习活跃度
   → 分析正确率变化
   
3. 准备教学调整建议
```

### **场景 2: 发现需要帮助的学生**

```
1. 打开"学生"标签页
2. 筛选特定班级
3. 查看排行榜底部学生
4. 点击展开查看详情
5. 关注:
   - 练习次数是否太少？
   - 最高和最低正确率差距大吗？
   - 本周是否有练习？
```

### **场景 3: 班级对比分析**

```
1. 打开"班级"标签页
2. 对比各班级:
   - 平均正确率
   - 人均练习次数
   - 总体参与度
3. 发现优秀班级的特点
4. 帮助落后班级改进
```

### **场景 4: 评估教学效果**

```
1. 打开"趋势"标签页
2. 查看正确率趋势
   - 上升 → 教学有效
   - 下降 → 需要调整
   - 平稳 → 巩固阶段
3. 结合难度分析
   - 学生是否在挑战更高难度？
   - 各难度的掌握情况如何？
```

## 💡 使用技巧

### **技巧 1: 定期查看**
建议：
- 每周一查看上周数据
- 每月初进行月度总结
- 期中期末重点分析

- 表扬进步大的学生
- 鼓励练习积极的学生
- 帮助遇到困难的学生

### **技巧 4: 班级激励**
- 公布班级排行
- 设置班级目标
- 组织班级间竞赛

## 🆘 常见问题

### **Q: 为什么看不到数据？**
**A:** 检查：
1. 学生是否完成过练习？
2. 练习时是否填写了姓名和班级？
3. 环境变量是否配置正确？

### **Q: 数据不准确？**
**A:** 
1. 点击"刷新"按钮
2. 清除浏览器缓存
3. 检查 Supabase 中的原始数据

### **Q: 某个学生找不到？**
**A:** 
1. 该学生是否填写了姓名？
2. 班级筛选是否选对？
3. 该学生是否真的有练习记录？

### **Q: 图表显示不正常？**
**A:** 
1. 尝试刷新页面
2. 检查浏览器是否支持（建议 Chrome/Edge）
3. 检查是否有足够的数据（至少需要几条记录）
