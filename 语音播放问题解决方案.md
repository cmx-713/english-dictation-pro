# 语音播放问题解决方案

## 🔍 **问题原因**

系统使用的是 **Web Speech API (Text-to-Speech)**，这是浏览器内置的文字转语音功能。

### **核心技术：**
```javascript
window.speechSynthesis.speak(utterance);
```

## 🐛 **常见问题和解决方案**

### **问题 1: 点击播放无反应**

**原因：**
- 浏览器阻止自动播放音频
- 需要用户手势触发（点击、触摸）

**解决：**
✅ 已解决 - 通过按钮点击触发

---

### **问题 2: 提示"未找到英文语音"**

**原因：**
- 系统未安装英文语音包
- 浏览器未加载语音列表

**解决方案：**

#### **Windows 系统：**
1. 打开 **设置** → **时间和语言** → **语音**
2. 添加 **English (United States)** 语音
3. 重启浏览器

#### **macOS 系统：**
1. 打开 **系统偏好设置** → **辅助功能** → **语音内容**
2. 选择英文语音（如 Samantha, Alex）
3. 重启浏览器

#### **Android 系统：**
1. 打开 **设置** → **系统** → **语言和输入法** → **文字转语音输出**
2. 安装 **Google Text-to-Speech**
3. 下载英语语音包

---

### **问题 3: 语音列表加载缓慢**

**原因：**
- `getVoices()` 异步加载
- 某些浏览器需要时间

**解决：**
✅ 已解决 - 代码中添加了：
- `onvoiceschanged` 事件监听
- 延迟重试机制
- 加载提示

---

### **问题 4: 长文本播放中断**

**原因：**
- Web Speech API 对文本长度有限制
- 约 32KB 或更少

**表现：**
- 超长文本播放不完整
- 中途停止

**解决方案：**
```typescript
// 自动分段播放（可选优化）
if (text.length > 5000) {
  // 分段处理
}
```

---

### **问题 5: Firefox 首次无法播放**

**原因：**
- Firefox 对用户手势要求更严格

**解决：**
✅ 已解决 - 确保通过按钮点击触发

---

### **问题 6: 微信浏览器限制**

**原因：**
- 微信浏览器对 Web API 有限制

**解决方案：**
```
点击右上角 "..." → 在浏览器中打开
```

## 🆘 **常见错误代码**

| 错误代码 | 含义 | 解决方案 |
|---------|------|----------|
| `not-allowed` | 浏览器阻止播放 | 检查权限设置 |
| `network` | 网络错误 | 检查网络连接 |
| `synthesis-unavailable` | TTS 不可用 | 检查语音包 |
| `synthesis-failed` | 合成失败 | 重启浏览器 |
| `audio-busy` | 音频忙碌 | 等待或刷新 |

## 🎓 **技术细节**

### **Web Speech API 限制：**
- 最大文本长度：约 32KB
- 语速范围：0.1 - 10
- 音量范围：0 - 1
- 音调范围：0 - 2

### **浏览器差异：**
- Chrome: 使用系统 TTS 引擎
- Safari: 使用 Siri 引擎
- Firefox: 使用 Speech Dispatcher (Linux)
- Edge: 使用 Windows SAPI
