## 📊 数据表结构（精简版）

```
practice_records 表：
├── id (UUID) - 唯一标识
├── created_at (TIMESTAMP) - 练习时间
├── student_name (TEXT) ⭐ 学生姓名
├── class_name (TEXT) ⭐ 班级名称
├── raw_text (TEXT) - 原文
├── results (JSONB) - 详细结果
├── accuracy_rate (DECIMAL) ⭐ 正确率
├── perfect_sentences (INT) ⭐ 完美句数
├── total_sentences (INT) ⭐ 总句数
├── total_words (INT) ⭐ 总单词数
├── difficulty_level (TEXT) ⭐ 难度等级
└── input_method (TEXT) ⭐ 输入方式
```

**只有12个核心字段，全部都有实际用途！**

## 📈 实用查询示例

### 查询 1: 查看某个学生的所有练习记录

```sql
SELECT 
  created_at,
  accuracy_rate,
  perfect_sentences,
  total_sentences,
  total_words,
  difficulty_level,
  LEFT(raw_text, 50) as text_preview
FROM practice_records
WHERE student_name = '张三'
ORDER BY created_at DESC;
```

### 查询 2: 班级整体表现

```sql
SELECT * FROM class_stats;
```

输出示例：
```
class_name | student_count | total_practices | avg_accuracy | total_words
-----------|--------------|-----------------|--------------|-------------
高一(3)班   | 25           | 156             | 87.5         | 2340
高一(2)班   | 23           | 142             | 85.3         | 2100
```

### 查询 3: 学生排行榜

```sql
SELECT 
  student_name,
  class_name,
  total_practices,
  avg_accuracy,
  total_words_practiced
FROM student_summary
ORDER BY avg_accuracy DESC
LIMIT 10;
```

### 查询 4: 找出需要帮助的学生

```sql
SELECT 
  student_name,
  class_name,
  avg_accuracy,
  total_practices,
  recent_practices
FROM student_summary
WHERE avg_accuracy < 70
ORDER BY avg_accuracy;
```

### 查询 5: 每日练习统计

```sql
SELECT * FROM daily_stats
WHERE practice_date >= CURRENT_DATE - INTERVAL '7 days';
```

### 查询 6: 查看某次练习的详细错误

```sql
SELECT 
  student_name,
  raw_text,
  results,  -- 这是 JSON 格式，包含每句的详细对比
  accuracy_rate,
  created_at
FROM practice_records
WHERE id = 'xxx-your-record-id-xxx';
```

**解析 results JSON：**

## 💡 使用场景

### 1. 个人学习追踪

**学生可以：**
- 查看自己的练习历史
- 了解进步趋势
- 发现薄弱环节

**SQL 查询：**
```sql
SELECT 
  DATE(created_at) as date,
  accuracy_rate,
  total_words
FROM practice_records
WHERE student_name = '你的姓名'
ORDER BY created_at;
```

### 2. 班级管理

**老师可以：**
- 查看班级整体表现
- 找出需要帮助的学生
- 监控学习活跃度

**使用视图：**
```sql
-- 班级概览
SELECT * FROM class_stats 
WHERE class_name = '高一(3)班';

-- 学生详情
SELECT * FROM student_summary
WHERE class_name = '高一(3)班'
ORDER BY avg_accuracy DESC;
```

### 3. 教学研究

**分析：**
- 不同难度素材的完成情况
- 学习时间分布规律
- 输入方式对效果的影响

**SQL 查询：**
```sql
-- 难度分析
SELECT * FROM difficulty_stats;

-- 输入方式对比
SELECT 
  input_method,
  COUNT(*) as count,
  ROUND(AVG(accuracy_rate), 1) as avg_accuracy
FROM practice_records
GROUP BY input_method;
```

## 🔧 故障排查

### 问题 1: 学生信息不保存

**原因：** 浏览器禁用了 localStorage

**解决：**
- 检查浏览器设置
- 尝试无痕模式
- 清除浏览器缓存后重试

### 问题 2: 数据没有上传到 Supabase

**检查：**
1. 打开浏览器控制台（F12）
2. 查看是否有错误信息
3. 检查 Supabase 环境变量是否正确配置

**验证环境变量：**
- 在 Netlify 后台查看：
  - `VITE_SUPABASE_URL`
  - `VITE_SUPABASE_ANON_KEY`

### 问题 3: SQL 执行失败

**常见原因：**
- 表名不匹配（确认是 `practice_records`）
- 字段已存在（可以安全忽略 "already exists" 警告）
- 权限不足（确认你有表的修改权限）
